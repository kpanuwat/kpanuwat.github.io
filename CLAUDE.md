# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Site

Jekyll static site using `jekyll-theme-chirpy` (~7.2), deployed to GitHub Pages via `.github/workflows/pages-deploy.yml` on push to `main`.

## Commands

```bash
bundle exec jekyll serve          # local dev server (http://localhost:4000)
bundle exec jekyll serve --drafts # include _drafts/
bundle exec jekyll build          # one-off build to _site/
JEKYLL_ENV=production bundle exec jekyll build  # production build (matches CI)
```

Marp (local preview):
```bash
npx -p @marp-team/marp-cli@4.4.0 marp -I _slides/ -o slides/ --html
```

No test suite. CI runs Marp then `bundle exec jekyll build --destination "_site"` with `JEKYLL_ENV=production`.

## Architecture

### Content layers

| Layer | Path | Purpose |
|-------|------|---------|
| Posts | `_posts/YYYY-MM-DD-slug.md` | Main content; date must be ≤ today or Jekyll suppresses it |
| Topic hubs | `topics/<name>.md` | One hub per content domain (Engineering, Tools, etc.) |
| Nav tabs | `_tabs/<name>.md` | Top nav items; `order:` controls position |
| Series registry | `_data/series.yml` | Register a series before using `series:` in post frontmatter |
| Slide decks | `_slides/<slug>.md` | Marp source; rendered to `slides/<slug>.html` by CI |

### Custom layouts (override Chirpy gem defaults)

- `_layouts/topic-hub.html` — renders `featured_posts` list + all posts that match `hub_category`; wraps Chirpy's `page` layout
- `_layouts/topics-index.html` — index of all topic hubs

### Custom includes

- `_includes/series-nav.html` — auto-injected when post frontmatter has `series:`; reads `_data/series.yml` for display title/description and builds ordered nav with prev/next
- `_includes/slide-link.html` — renders a "View Slides" link when post frontmatter has `slides: true`; links to `/slides/{{ page.slug }}.html`

## Post conventions

Use `_posts/.post-template.md` as the canonical template.

Required frontmatter:
```yaml
title: "..."
date: YYYY-MM-DD HH:MM:SS +0700   # future dates are suppressed by Jekyll
categories: [CategoryName]         # must match a topic hub's hub_category to appear there
tags: [lowercase, hyphen-ok]
```

Optional frontmatter: `series`, `slides`, `pin`, `math`, `mermaid`, `image`.

Default post structure: `## TL;DR` → content sections → `## Key Takeaways`.

## Adding a Topic Hub

1. Create `topics/<slug>.md` with frontmatter:
   ```yaml
   layout: topic-hub
   title: ...
   description: ...
   hub_category: CategoryName     # must match categories: value used in posts
   permalink: /topics/<slug>/
   featured_posts: [post-slug-1]  # post filename without date and .md extension
   ```
2. Add a `_tabs/<slug>.md` nav tab if the topic should appear in top nav.

## Adding a Series

1. Register in `_data/series.yml`:
   ```yaml
   - name: series-slug
     title: "Display Title"
     description: "Shown in nav block"
   ```
2. Add `series: series-slug` to each post's frontmatter.
3. `_includes/series-nav.html` renders automatically — no layout changes needed.

## Adding a Slide Deck

> **Visual authoring rules** (SVG-first, no bullet defaults, CSS token discipline, 9 SVG patterns)
> live in the `marp-deck` skill. The skill auto-runs after each new/updated post —
> manual `marp` invocations should also load it. See `~/.claude/skills/marp-deck/SKILL.md`.

Slides live in `_slides/` (Jekyll ignores `_`-prefixed dirs) and are rendered by Marp CLI in CI to `slides/` (served at `/slides/<slug>.html`).

1. Write `_slides/<slug>.md` — use `_slides/.deck-template.md` as starting point.
   Required Marp frontmatter:
   ```yaml
   ---
   marp: true
   theme: default
   paginate: true
   ---
   ```
   Use `---` on its own line to separate slides.

2. Add `slides: true` to the corresponding post's frontmatter.
   `_includes/slide-link.html` renders the link automatically; no layout changes needed.

3. Preview locally:
   ```bash
   npx -p @marp-team/marp-cli@4.4.0 marp _slides/<slug>.md -o slides/<slug>.html --html
   bundle exec jekyll serve
   # Visit http://localhost:4000/slides/<slug>.html
   ```

4. Commit only `_slides/<slug>.md` — never commit `slides/*.html` (generated by CI).

**Known limitation:** if you delete `slides/` locally and run `jekyll serve` without re-running Marp, slide URLs return 404 until next build. `keep_files: [slides]` in `_config.yml` prevents Jekyll from cleaning the dir, but won't restore missing files.
