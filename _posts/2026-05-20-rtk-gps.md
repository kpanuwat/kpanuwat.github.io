## TL;DR

RTK (Real-Time Kinematic) คือเทคนิคเพิ่มความแม่นยำ GPS/GNSS จากระดับ 2–5 เมตร → **1–2 ซม.** โดยใช้ base station ส่ง correction แบบ real-time ให้ rover รับและชดเชย error ร่วมกัน

---

## Background / Why this matters

GPS ปกติที่ใช้กันทั่วไปในโทรศัพท์หรือรถยนต์มีความแม่นยำอยู่ที่ประมาณ 2–5 เมตร เพราะสัญญาณดาวเทียมถูกรบกวนจากหลายแหล่ง ได้แก่ ชั้นบรรยากาศ (ionosphere, troposphere), clock error ของดาวเทียม, และ multipath (สัญญาณสะท้อนจากอาคาร)

สำหรับงานที่ต้องการความแม่นยำสูง เช่น การสำรวจที่ดิน, เกษตรกรรมแม่นยำ, การบินโดรน, หรือระบบ autopilot ความแม่นยำระดับเมตรไม่เพียงพอ

RTK แก้ปัญหานี้ด้วยการใช้ receiver 2 ตัวพร้อมกัน ตัวหนึ่งอยู่กับที่ (base) รู้พิกัดแน่นอน อีกตัวเคลื่อนที่ได้ (rover) โดยทั้งคู่รับสัญญาณดาวเทียมชุดเดียวกันในเวลาเดียวกัน ดังนั้น error จากบรรยากาศและดาวเทียมจะเหมือนกันทั้งคู่ base จึงคำนวณ correction แล้วส่งให้ rover ชดเชยได้ทันที

## Deep Dive

### หลักการ: Carrier Phase vs. Code Phase

GPS ทั่วไปวัดระยะด้วย **pseudorange** (code-phase) — ดูว่าสัญญาณใช้เวลาเดินทางนานแค่ไหน ความละเอียดจำกัดที่ความถี่ของ code (~1.023 MHz)

RTK ใช้ **carrier-phase measurement** แทน — วัด phase ของคลื่นพาหะ L1 (1575.42 MHz) ซึ่งเปลี่ยน phase ถี่กว่า code ถึง ~1,000 เท่า ทำให้ค่าผิดพลาด ±1% ของ cycle ตรงกับระยะผิดพลาดเพียง ~1.9 มม. เท่านั้น

ปัญหาคือ receiver ไม่รู้ว่าในช่วงเวลาที่ผ่านมามีคลื่นเต็ม cycle กี่ครั้ง (integer ambiguity) RTK ต้องแก้ปัญหา ambiguity นี้ก่อน จึงจะได้ความแม่นยำระดับ cm

### สถานะการทำงาน 3 ระดับ

| สถานะ | ความแม่นยำ | ความหมาย |
|-------|-----------|----------|
| **FIX** | 1–2 ซม. horizontal | แก้ integer ambiguity สำเร็จ |
| **FLOAT** | 10–50 ซม. | รับ correction แล้ว แต่ ambiguity ยังไม่ resolved |
| **SINGLE** | 2–5 ม. | ไม่มี correction (GPS ปกติ) |

งานที่ต้องการความแม่นยำสูงจะรับได้เฉพาะสถานะ **FIX** เท่านั้น

### Base Station กับ Rover

```
ดาวเทียม GPS/GLONASS/Galileo/BeiDou
        │         │
        ▼         ▼
   [Base Station] ←── รู้พิกัดแน่นอน
        │               คำนวณ error
        │  correction    
        │  (radio/4G/NTRIP)
        ▼
    [Rover] ──► ชดเชย error ► ได้พิกัด cm-level
```

**Base station** ตั้งอยู่กับที่ที่พิกัดที่รู้แน่นอน วัดสัญญาณดาวเทียม แล้วเปรียบเทียบกับตำแหน่งจริง ความต่างคือ error → ส่งเป็น correction ไปให้ rover

**Rover** รับทั้งสัญญาณดาวเทียมโดยตรง และ correction จาก base นำมาหักลบกัน

### วิธีส่ง Correction

| วิธี | เหมาะกับ | ข้อดี/ข้อเสีย |
|-----|---------|--------------|
| **Radio (UHF)** | งานสนามที่ไม่มี internet | ไม่ต้องใช้ SIM แต่ระยะจำกัด ~10 กม. |
| **NTRIP** | มี 4G/WiFi | ใช้เครือข่าย reference station สาธารณะ/เช่าได้ |
| **4G cellular** | ทุกที่ที่มีสัญญาณ | สะดวก แต่ต้องจ่ายค่า subscription |

**NTRIP** (Networked Transport of RTCM via Internet Protocol) คือ standard ที่ให้ rover เชื่อมต่อกับ NTRIP caster ผ่าน internet และรับ correction โดยไม่ต้องตั้ง base เอง — มีหน่วยงานรัฐและเอกชนหลายแห่งให้บริการ

### ระยะ Baseline ที่ยอมรับได้

- Single-band receiver: ไม่เกิน ~10–20 กม. จาก base
- Multi-band receiver: รับได้ถึง ~30–50 กม.
- เกินนั้น → ใช้ **Network RTK** หรือ **VRS** (Virtual Reference Station) ที่รวมข้อมูลจากหลาย base พร้อมกัน

### RTK vs. PPK

| | **RTK** | **PPK** (Post-Processing Kinematic) |
|--|---------|--------------------------------------|
| เวลาประมวลผล | Real-time | หลังภารกิจ |
| ใช้กับ | สำรวจ, machine control | Drone mapping, ภาพถ่ายทางอากาศ |
| ข้อดี | เห็นผลทันที, stake-out ได้ | ไม่ต้องส่ง correction แบบ live |
| ข้อเสีย | ต้องมี link ตลอดเวลา | ต้องประมวลผลหลังงาน |

ทั้งสองใช้คณิตศาสตร์เดียวกัน แตกต่างแค่เวลาที่แก้ ambiguity

## User Guide (Step-by-Step)

### Setup แบบง่าย: ใช้ NTRIP (ไม่ต้องตั้ง base เอง)

**สิ่งที่ต้องมี:**
- RTK receiver (เช่น Emlid Reach RS2+, u-blox ZED-F9P)
- SIM card ที่มี 4G
- NTRIP account จากผู้ให้บริการ (ไทย: RTSD, GISTDA, เอกชน)

**ขั้นตอน:**
1. เปิด receiver → ตั้งค่า NTRIP host, port, mountpoint, username/password
2. รอสถานะเปลี่ยนจาก SINGLE → FLOAT → FIX (ปกติ 30–120 วินาที)
3. เมื่อ FIX → เริ่มเก็บ waypoint หรือ stakeout ได้
4. ตรวจสอบ PDOP (Positional Dilution of Precision) — ค่าต่ำกว่า 3 คือ geometry ดี

**Troubleshooting:**
- ค้างที่ FLOAT นาน → ลอง reboot receiver หรือตรวจสอบ satellite count (ควรมีอย่างน้อย 6 ดวง)
- สูญเสีย FIX กลางงาน → multipath (อยู่ใกล้อาคาร/ต้นไม้) หรือ baseline ไกลเกินไป
- NTRIP เชื่อมไม่ได้ → ตรวจสอบ mountpoint ที่ถูกต้องสำหรับพื้นที่ทำงาน

## Caveats / Limits

- **Multipath**: อาคาร ต้นไม้ สิ่งกีดขวางทำให้สัญญาณสะท้อน → accuracy แย่ลง
- **Sky obstruction**: ต้องเห็นท้องฟ้าเปิดโล่ง ทำงานในอาคารหรือป่าทึบไม่ได้
- **Initialization time**: ต้องรอ FIX lock ก่อน บางครั้งนาน 2–5 นาที
- **Baseline limit**: ยิ่งไกล base ยิ่ง accuracy แย่ (8mm + 1ppm per km)
- **Single-band vs. multi-band**: receiver ราคาถูก (single-band) จะ struggle เมื่อ baseline > 10–15 กม. หรือในสภาพ ionosphere แปรปรวน
- **Cost**: receiver คุณภาพดี ($500–$5,000+) + NTRIP subscription ($50–$200/เดือน)
- **ไม่ใช่ real-time ทุกกรณี**: ถ้าสัญญาณ correction ขาด → ตกกลับมา FLOAT/SINGLE โดยอัตโนมัติ

## Key Takeaways

- RTK = GPS + correction จาก base station → ความแม่นยำจาก ~3 ม. → **1–2 ซม.**
- กุญแจสำคัญคือ **carrier-phase measurement** แทน code-phase — ละเอียดกว่า ~1,000 เท่า
- สถานะ **FIX** = ambiguity resolved = cm accuracy; **FLOAT** = correction มีแต่ยังไม่ full precision
- ส่ง correction ได้ 3 วิธี: radio, 4G, หรือ NTRIP (internet-based network station)
- Baseline ควรอยู่ภายใน 20–30 กม. จาก base (multi-band) หรือ ~10 กม. (single-band)
- RTK = real-time; PPK = post-processing — ใช้ math เดียวกัน แตกต่างที่เวลาประมวลผล
- ข้อจำกัดหลัก: multipath, sky obstruction, baseline distance, initialization time
