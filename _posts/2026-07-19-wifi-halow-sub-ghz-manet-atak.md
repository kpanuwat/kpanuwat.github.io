## TL;DR

Wi-Fi HaLow (IEEE 802.11ah) เป็น Wi-Fi ย่านความถี่ต่ำกว่า 1GHz ที่แลกความเร็วเอากับระยะทางและพลังงานต่ำ — เอามาผูกกับ Raspberry Pi + ซอฟต์แวร์ทหารโอเพนซอร์สอย่าง ATAK ก็สร้างเครือข่าย MANET (mobile ad-hoc network) ที่ส่ง video streaming แบบ P2P ได้จริงโดยไม่ง้อค่ายมือถือหรือดาวเทียม ด้วยฮาร์ดแวร์รวมกันไม่ถึง 100 ดอลลาร์

## Background / Why this matters

ทุกวันนี้ Wi-Fi ถูกผูกไว้กับสถานที่ — เราติดตั้งเราเตอร์ในบ้านหรือออฟฟิศ แล้วให้มันครอบคลุมพื้นที่จำกัดแค่นั้น พอออกนอกรัศมีสัญญาณ ก็ต้องพึ่งเครือข่ายมือถือ (ซึ่งต้องมีเสาสัญญาณ) หรือดาวเทียม (ซึ่งมีค่าใช้จ่ายสูงและต้องพึ่งผู้ให้บริการ) วิดีโอต้นทางของโพสต์นี้ ["You've Never Seen Wi-Fi Like This"](https://youtu.be/0XhqjaGmKiw) ตั้งคำถามว่า ถ้า Wi-Fi เคลื่อนที่ไปกับเราได้ — ขึ้นเขา ลงเรือคายัค ไปพื้นที่ห่างไกล — โดยไม่ต้องมีเสาส่งสัญญาณหรือดาวเทียมเลยล่ะ?

คำตอบที่วิดีโอนำเสนอคือการผสมสามอย่างเข้าด้วยกัน: (1) Wi-Fi HaLow ซึ่งเป็นมาตรฐาน Wi-Fi ย่านความถี่ต่ำที่แลกความเร็วเพื่อระยะทางไกลขึ้นและใช้พลังงานน้อยลง (2) แนวคิด MANET (Mobile Ad-hoc Network) ที่ทหารใช้มานานหลายปีในเทคโนโลยีอย่าง Persistent Systems MPU-5 ราคาต่อเครื่องประมาณ 15,000-20,000 ดอลลาร์ และ (3) ATAK (Android Team Awareness Kit) ซอฟต์แวร์ situational-awareness ของกองทัพสหรัฐฯ ที่ถูกเปิดซอร์สให้พลเรือนใช้ได้ฟรีตั้งแต่ปี 2020

ประเด็นที่ทำให้เรื่องนี้น่าสนใจไม่ใช่แค่ technical demo แต่คือการ "ลดการผูกขาด" — อุปกรณ์ที่เคยล็อกอยู่ในงบกลาโหมและสัญญาทหาร ตอนนี้ประกอบได้จากชิ้นส่วนโอเพนซอร์สราคาต่ำกว่า 100 ดอลลาร์ ใครก็ซื้อได้ ไม่มี subscription ไม่มี cloud login ไม่มีใครมาตัดสัญญาณเราได้

## Deep Dive

### Wi-Fi HaLow (802.11ah) คืออะไร

Wi-Fi HaLow คือชื่อทางการค้าของมาตรฐาน IEEE 802.11ah ซึ่งเผยแพร่ในปี 2017 เป็นส่วนขยายของมาตรฐาน Wi-Fi เดิม แต่ทำงานในย่านความถี่ sub-1GHz (902-928MHz ในอเมริกาเหนือ, 863-868MHz ในยุโรป) แทนที่จะเป็น 2.4GHz/5GHz/6GHz แบบ Wi-Fi ทั่วไป — ผลคือคลื่นทะลุทะลวงสิ่งกีดขวางได้ดีกว่าและเดินทางได้ไกลกว่ามาก โดยระยะครอบคลุมอยู่ที่ประมาณ 100 เมตรถึง 1 กิโลเมตร ขึ้นกับสภาพแวดล้อม[^1]

### ความถี่ Sub-GHz กับ trade-off ระยะ-ความเร็ว

อัตราการส่งข้อมูลของ HaLow มีช่วงกว้างมาก ตั้งแต่ 150 kbps (แบนด์วิดท์ช่องสัญญาณ 1MHz) ไปจนถึง 86.7 Mbps (แบนด์วิดท์ 16MHz)[^1] — นี่คือกฎฟิสิกส์พื้นฐานของวิทยุ: ยิ่งใช้แบนด์วิดท์แคบ (channel width ต่ำ) ยิ่งได้ระยะไกลขึ้นแต่ throughput ต่ำลง ในวิดีโอต้นทาง ผู้สร้างพูดถึงเรื่องนี้ตรงๆ ว่าถ้าไม่จำเป็นต้องสตรีมวิดีโอ (bit rate สูง) ก็สามารถลดไปใช้ channel width แคบลง เช่น 1-4 MHz เพื่อดันระยะทางให้ไกลขึ้นได้อีก

อีกตัวแปรสำคัญคือกำลังส่ง (transmit power) — กฎเกณฑ์อนุญาตสูงสุดคือ 1 วัตต์ หรือเทียบเท่า 30 dBm แต่ฮาร์ดแวร์ที่ใช้สาธิตจริง (Seeed Studio HaLow HAT รัน firmware OpenWrt ค่า default) ส่งที่ 21 dBm เท่านั้น ผู้สร้างวิดีโอกล่าวว่ามีคนติดต่อมาเสนอ firmware ที่ปรับแต่งเองให้ดันกำลังส่งขึ้นไปถึง 27 dBm ได้ — ยังห่างจากค่าสูงสุด 30 dBm ที่กฎหมายอนุญาต

### MANET คืออะไร ต่างจาก Wi-Fi ทั่วไปอย่างไร

MANET (Mobile Ad-hoc Network) คือเครือข่ายที่ทุกอุปกรณ์ทำหน้าที่เป็น "โหนด" (node) เท่าเทียมกัน ไม่มี master node ศูนย์กลาง ไม่มีเสาสัญญาณ ไม่มีดาวเทียม — แต่ละโหนด route ข้อมูลแบบ peer-to-peer ให้กันเองแบบเรียลไทม์ และเครือข่ายยัง "self-healing" ได้ต่อแม้บางโหนดจะหลุดออกจากวง ต้นแบบทางทหารที่วิดีโออ้างถึงคือ Persistent Systems MPU-5 ซึ่งรองรับจำนวน hop และจำนวนโหนดแบบไม่จำกัด ระยะห่างสูงสุดระหว่างโหนดถึง 130 ไมล์ ให้ throughput จริงสูงสุด 100 Mbps ด้วยกำลังส่งสูงสุด 10W และผ่านมาตรฐาน MIL-STD-810G กับกันน้ำระดับ IP68[^5] — แต่ราคาต่อเครื่องอยู่ที่ 15,000-20,000 ดอลลาร์ (ตามที่ผู้สร้างวิดีโออ้างอิง รวมซอฟต์แวร์ เสาอากาศ และค่าไลเซนส์)

### ฮาร์ดแวร์ในสาธิต: Raspberry Pi + Seeed Studio HaLow HAT + XIAO

ฝั่ง gateway/access point: Raspberry Pi ต่อกับ Seeed Studio Wi-Fi HaLow HAT (ใช้ชิป Quectel FGH100M-H รองรับย่าน 902-928MHz ตามมาตรฐาน 802.11ah ระยะครอบคลุมสูงสุด 1km และ PHY rate สูงสุด 32.5 Mbps รองรับ WPA3/AES)[^3] เสียบสาย Ethernet เข้ากับเราเตอร์หลัก (ในวิดีโอใช้ UniFi Express) วิธีนี้ทำให้ทุกอุปกรณ์ที่คุยผ่านคลื่น HaLow เข้าถึง IP network เดียวกับเราเตอร์หลักได้ทันที โดยไม่ต้องตั้งค่าเน็ตเวิร์กแยก

ฝั่ง client เคลื่อนที่: บอร์ด Seeed Studio XIAO (ราคาราวๆ 6 ดอลลาร์) ต่อกับโมดูล HaLow, กล้อง FPV, เสาอากาศระยะไกล และแบตเตอรี่ 18650 พร้อมพอร์ต USB-C ในตัว — จุดที่น่าทึ่งคือบอร์ดนี้มี RAM/storage แทบไม่มีเลย (ระดับ MCU ไม่ใช่ SBC เต็มรูปแบบ) แต่ยังสตรีมวิดีโอผ่าน HTTP ข้ามระยะไกลได้จริง เพราะ HaLow ให้ IP networking แบบเต็มรูปแบบ ต่างจาก LoRa/Meshtastic ที่ throughput ต่ำเกินกว่าจะส่งวิดีโอได้

### ATAK: ซอฟต์แวร์จากกองทัพที่กลายเป็นโอเพนซอร์ส

ในวิดีโอต้นทาง ผู้บรรยายออกเสียงชื่อซอฟต์แวร์นี้คล้าย "ATAC" แต่ชื่อจริงคือ **ATAK (Android Team Awareness Kit)** — เครื่องมือ situational-awareness ที่กองทัพสหรัฐฯ ใช้แสดงแผนที่ ตำแหน่งทีม ข้อความ และ overlay ภาพจากโดรนแบบเรียลไทม์ วันที่ 19 สิงหาคม 2020 กระทรวงกลาโหมสหรัฐฯ (ผ่าน Defense Digital Service) เปิดซอร์สโค้ดเวอร์ชันพลเรือน (ATAK-CIV) ให้ดาวน์โหลดฟรีบน GitHub ภายใต้ไลเซนส์ GPLv3[^2] ดาวน์โหลดได้ทั้งจาก Google Play Store หรือโดยตรงจาก tak.gov โดยไม่ต้องผ่านตัวกลางใดๆ — ในวิดีโอ ผู้สร้างติดตั้ง APK ตรงจาก tak.gov บนมือถือ Android ราคาถูก แล้วสาธิตทั้งการส่งข้อความและสตรีมวิดีโอสดผ่านเครือข่าย HaLow เข้าสู่แอป ATAK ได้สำเร็จ

### B.A.T.M.A.N. และ 802.11s: อนาคตของ mesh routing บน HaLow

การสาธิตในวิดีโอยังไม่ใช่ MANET แบบเต็มรูปแบบ เพราะยังไม่มี mesh routing จริง — เป็นแค่ point-to-point ระหว่างสอง node เท่านั้น ผู้สร้างวิดีโอระบุแผนสำหรับวิดีโอถัดไปว่าจะติดตั้ง **B.A.T.M.A.N.** (Better Approach To Mobile Ad-hoc Networking) ซึ่งเป็น routing protocol แบบ proactive ที่ชุมชน Freifunk พัฒนาขึ้นมาแทน OLSR สำหรับเครือข่าย mesh ขนาดใหญ่[^4] ควบคู่กับมาตรฐาน 802.11s (มาตรฐาน mesh ของ IEEE เอง ซึ่งกำหนด Hybrid Wireless Mesh Protocol หรือ HWMP เป็นค่าเริ่มต้น) — ข้อควรรู้คือ B.A.T.M.A.N. ไม่ได้ compliant กับ 802.11s โดยตรง แต่ทำงานร่วมกันได้ผ่านการตั้งค่าบน OpenWrt (batman-adv)[^4]

### เทียบกับ Meshtastic/LoRa

วิดีโอเปรียบเทียบ HaLow กับ Meshtastic (ที่ใช้ชิป LoRa อย่าง T1000-E) ตรงๆ ว่า LoRa ให้ระยะไกลกว่ามากในบางกรณี แต่ **ไม่ใช่ true IP networking** — เหมาะกับข้อความสั้นและ telemetry เท่านั้น ส่งวิดีโอหรือสตรีมมิ่งไม่ได้ ขณะที่ HaLow แม้ระยะจะสั้นกว่า LoRa ในทางทฤษฎี แต่ให้ throughput พอสำหรับวิดีโอ lightweight และรองรับ IP stack เต็มรูปแบบ วิสัยทัศน์ในวิดีโอคือ MANET ในอนาคตจะรวมทั้ง 2.4GHz, 5GHz, HaLow และ LoRa (ผ่าน Meshtastic plugin) เข้าด้วยกัน โดยให้ HaLow รับหน้าที่ backbone ระยะไกล

### ผลการทดสอบภาคสนามจริง (Miami field test)

ผู้สร้างวิดีโอทดสอบจริงริมน้ำในไมอามี โดยติดกล้อง FPV ไว้กับต้นไม้ (node คงที่) แล้วขับรถออกไปพร้อม ATAK เปิดดูสตรีมสด (node เคลื่อนที่) — ผลคือสตรีมวิดีโอ bit-rate สูงยังคงใช้งานได้ต่อเนื่องจนถึงระยะประมาณ **1,500 ฟุต** ก่อนสัญญาณเริ่มขาด นี่คือระยะที่ได้จากฮาร์ดแวร์กำลังส่งต่ำกว่าค่าสูงสุดที่กฎหมายอนุญาต (21 dBm จาก 30 dBm) และยังไม่ได้ปรับ channel width ให้แคบลงเพื่อดันระยะทาง — ผู้สร้างวิดีโอสรุปว่านี่ไกลกว่าเราเตอร์ Wi-Fi ทั่วไปมาก แต่ยังไม่พอสำหรับงาน search-and-rescue จริงจัง จนกว่าจะปรับแต่ง firmware/channel width เพิ่มเติม

### ต้นทุนฮาร์ดแวร์ทั้งระบบ

ตามตัวเลขที่ผู้สร้างวิดีโอระบุ: Raspberry Pi ~$35, เสาอากาศ HaLow ~$5, ชิป HaLow ~$15, HAT ~$15 (ฝั่ง gateway รวมประมาณ $70) บวกกับบอร์ด XIAO ฝั่ง client ~$6 — เทียบกับ MPU-5 ระดับทหารที่ $15,000-20,000 ต่อเครื่อง ถือเป็นส่วนต่างที่ใหญ่มาก แม้จะเทียบ capability กันตรงๆ ไม่ได้ (MPU-5 แรงกว่ามาก ทั้งระยะ กำลังส่ง และการรับรองมาตรฐานทหาร) แต่ก็แสดงให้เห็นว่าแนวคิดพื้นฐานเดียวกันตอนนี้จับต้องได้ในระดับผู้บริโภคทั่วไปแล้ว

## User Guide (Step-by-Step)

หมายเหตุ: ขั้นตอนนี้สรุปจากสิ่งที่วิดีโอต้นทางสาธิตจริง เป็นแนวทางระดับ proof-of-concept ไม่ใช่คู่มือ production

1. **จัดหาฮาร์ดแวร์ฝั่ง gateway** — Raspberry Pi + Seeed Studio Wi-Fi HaLow HAT (ชิป Quectel FGH100M-H) + เสาอากาศ HaLow พร้อมฐานแม่เหล็ก
2. **แฟลชเฟิร์มแวร์ OpenWrt ของ Seeed Studio** ลงบน HaLow HAT แล้วเสียบ Ethernet เข้ากับเราเตอร์หลักที่มีอยู่ (2.4/5GHz) — ผลคือเราเตอร์หลักได้ "ขยาย" ให้รองรับคลื่น HaLow โดยอุปกรณ์ทุกตัวยังอยู่ใน IP network เดียวกัน
3. **ประกอบฝั่ง client เคลื่อนที่** — บอร์ด Seeed XIAO + โมดูล HaLow + กล้อง FPV + แบตเตอรี่ 18650 (พอร์ต USB-C ในตัว) ตั้งค่า SSID/password ให้ตรงกับฝั่ง gateway เพื่อ handshake กัน
4. **ติดตั้ง ATAK-CIV** บนอุปกรณ์ Android โดยดาวน์โหลด APK ตรงจาก tak.gov (ไม่ต้องผ่าน Google Play ก็ได้) แล้วตั้งค่า video feed ให้ชี้ไปที่ HTTP stream ของกล้อง FPV
5. **ทดสอบระยะ** — เปิดสตรีมค้างไว้แล้วเคลื่อน client ออกจาก gateway ทีละระยะ สังเกตว่าสตรีมยังลื่นอยู่จนถึงจุดไหน (ในการทดสอบภาคสนามจริงได้ประมาณ 1,500 ฟุตก่อนขาดสัญญาณ)
6. **ปรับแต่งเพื่อดันระยะเพิ่ม (ทางเลือก)** — ลด channel width ลง (เช่น จาก 8MHz เหลือ 1-4MHz) เพื่อแลก throughput กับระยะทาง หรือใช้ firmware ที่ปรับกำลังส่งขึ้นถึง 27 dBm แทนค่า default 21 dBm

## Caveats / Limits

- **นี่ไม่ใช่ MANET แบบเต็มรูปแบบ** — การสาธิตในวิดีโอต้นทางเป็นแค่ point-to-point ระหว่างสอง node ยังไม่มี mesh routing (B.A.T.M.A.N./802.11s) จริง ผู้สร้างวิดีโอเองยืนยันว่าจะทำ mesh เต็มรูปแบบในวิดีโอถัดไป
- **ระยะทางที่ทดสอบได้จริง (~1,500 ฟุต) ยังไม่พอสำหรับงาน search-and-rescue** ในสถานการณ์จริงที่ต้องการระยะเป็นกิโลเมตร ต้องปรับ firmware/channel width/กำลังส่งเพิ่มเติม และผลลัพธ์ยังขึ้นกับสภาพแวดล้อม (สิ่งกีดขวาง, สภาพอากาศ)
- **ข้อจำกัดด้านกฎหมาย** — กำลังส่งสูงสุดที่กฎหมายอนุญาตคือ 1W (30dBm) และย่านความถี่ 902-928MHz เป็นค่าสำหรับอเมริกาเหนือเท่านั้น ประเทศอื่นมีย่านความถี่ที่อนุญาตต่างกัน (เช่น ยุโรป 863-868MHz) ต้องตรวจสอบกฎระเบียบ RF ของแต่ละประเทศก่อนใช้งาน
- **ฮาร์ดแวร์ HaLow ยังหายาก/ไม่แพร่หลาย** — ชิปเซ็ตและอุปกรณ์รองรับ 802.11ah ยังมีผู้ผลิตน้อยราย เทียบกับ Wi-Fi มาตรฐาน 2.4/5GHz ที่หาซื้อได้ทั่วไป
- **Trade-off ระยะ vs throughput เป็นข้อจำกัดทางฟิสิกส์** ไม่ใช่บั๊ก — ยิ่งต้องการระยะไกล ยิ่งต้องยอมเสีย bandwidth และในทางกลับกัน
- **ความปลอดภัย**: แม้ ATAK-CIV จะรองรับ encryption และ HaLow HAT รองรับ WPA3/AES แต่การตั้งค่าที่ปลอดภัยต้องพิจารณาเรื่อง key management ของทั้งเครือข่าย HaLow และ ATAK server แยกกัน วิดีโอต้นทางไม่ได้ลงรายละเอียดการ hardening ความปลอดภัย

## Key Takeaways

- Wi-Fi HaLow (802.11ah) ทำงานที่ 902-928MHz (US) แลกความเร็ว (150kbps-86.7Mbps) เอาระยะทาง (100m-1km) และพลังงานต่ำ ต่างจาก Wi-Fi มาตรฐานที่ 2.4/5/6GHz
- MANET แบบทหาร (MPU-5) ราคา $15,000-20,000/เครื่อง แต่แนวคิดเดียวกันประกอบจากของโอเพนซอร์สได้ในราคารวมไม่ถึง $100
- ATAK (ไม่ใช่ "ATAC" ตามที่ออกเสียงในวิดีโอ) คือซอฟต์แวร์ situational-awareness ของกองทัพสหรัฐฯ ที่เปิดซอร์สให้พลเรือนใช้ฟรีตั้งแต่ปี 2020
- การสาธิตจริงทำระยะได้ ~1,500 ฟุต ด้วยกำลังส่งแค่ 21dBm จากค่าสูงสุดตามกฎหมาย 30dBm — ยังมีช่องให้พัฒนาต่อ
- HaLow ต่างจาก LoRa/Meshtastic ตรงที่ให้ true IP networking เต็มรูปแบบ รองรับวิดีโอสตรีมมิ่งได้ ไม่ใช่แค่ text/telemetry
- การสาธิตนี้ยังไม่ใช่ mesh routing เต็มรูปแบบ — ต้องรอ B.A.T.M.A.N. + 802.11s ในวิดีโอถัดไปของผู้สร้าง
- ทุก node ควบคุมโครงสร้างพื้นฐานเครือข่ายของตัวเองได้เต็มที่ ไม่มี subscription ไม่มี cloud login ไม่มีใครตัดสัญญาณได้
