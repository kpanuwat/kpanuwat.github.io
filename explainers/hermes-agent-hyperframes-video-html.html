02

เทคโนโลยีหลัก

HyperFrames: HTML → Video Pipeline

เขียน HTML → Headless Chrome จับทีละ frame → FFmpeg แปลงเป็น MP4 + มิกซ์เสียง → ได้วิดีโอที่ render ซ้ำได้ผลเหมือนกันทุกครั้ง

Animation Libraries ที่รองรับ

Adapter-Based

  • GSAP — timeline ต้องเริ่มด้วย { paused: true }
  • Lottie, Anime.js, WAAPI — seekable animations ทุกแบบ
  • Three.js — 3D Canvas effects
  • CSS Keyframes — ใช้ได้เลย ไม่ต้อง config เพิ่ม

ข้อได้เปรียบหลัก

Open-Source, No Fees

  • License: Apache 2.0 — ใช้เชิงพาณิชย์ได้ฟรี
  • ไม่มี per-render fee หรือ commercial threshold
  • Render ได้ใน CI/CD pipeline อัตโนมัติ
  • ใช้ร่วมกับ Claude Code, Cursor, Gemini CLI ได้ทันที
★ สาระสำคัญ: ใช้ Headless Chrome จับ frame ทีละช็อต แล้ว FFmpeg เชื่อมกัน — deterministic ทุก render ไม่มีความบังเอิญ
03

Agent ที่ขับเคลื่อน Pipeline

Hermes Agent: AI ที่เขียนวิดีโอแทนคุณ

Hermes Agent คือระบบ AI agent จาก Nous Research ที่รู้จัก HyperFrames ผ่าน skill ที่ติดตั้งได้ด้วยคำสั่งเดียว Agent ทำหน้าที่ทั้ง orchestrate script, เขียน HTML composition, จัดการ animation timing, และ render ออกมาเป็น MP4 โดยอัตโนมัติ

Layer 1

Hermes Skill

Task orchestration — รับ prompt, วางแผน script, กำหนดโครงสร้าง scene, loop แก้ไขจนได้ผลลัพธ์ที่ต้องการ

Layer 2

HyperFrames Model

HTML + timing schema — กำหนด data-start, data-duration, data-track-index สำหรับแต่ละ clip และ element

Layer 3

HyperFrames CLI

Build → Preview → Render — รัน lint, validate, preview, render ผ่าน command line โดย Agent

# ติดตั้ง HyperFrames skill ใน Hermes Agent / Claude Code
npx skills add heygen-com/hyperframes

# เริ่มโปรเจกต์ใหม่
npx hyperframes init my-video --example swiss-grid

# Preview ใน browser (hot-reload)
npx hyperframes preview
★ สาระสำคัญ: ติดตั้ง skill ครั้งเดียว — Agent เขียน HTML, lint, preview, และ render ได้ครบในลูปเดียว ไม่ต้องเปิด GUI ใดๆ
04

คู่มือปฏิบัติ

Workflow: จาก Prompt ถึง MP4 ใน 7 ขั้นตอน

  1. 1
    Plan — วางแผน narrative arc และ visual identity

    กำหนดเป้าหมายวิดีโอ, กลุ่มเป้าหมาย, จำนวน scene, และ hero frame ก่อนเขียนโค้ด

  2. 2
    Scaffold — สร้างโครงโปรเจกต์ด้วย template

    รัน npx hyperframes init my-video --example <template> เลือก template ที่เหมาะกับเนื้อหา เช่น swiss-grid (corporate) หรือ play-mode (social)

  3. 3
    Layout — สร้าง HTML/CSS สำหรับ hero frame ก่อน

    เริ่มจาก static layout ของ frame ที่สำคัญที่สุด ให้ดูดีใน browser ก่อนเพิ่ม animation

  4. 4
    Animate — เพิ่ม GSAP timeline (seekable เท่านั้น)

    ทุก timeline ต้องเริ่มด้วย gsap.timeline({ paused: true }) และ register บน window.__timelines — ห้ามใช้ repeat: -1

  5. 5
    Transitions — ใส่ shader transitions ระหว่าง scene

    ติดตั้ง catalog block เช่น npx hyperframes add flash-through-white หรือ liquid-wipe เพื่อป้องกัน jump cut

  6. 6
    Media — เพิ่ม audio, captions, TTS

    รัน npx hyperframes tts สร้าง voiceover หรือ npx hyperframes transcribe สร้าง caption จาก audio ที่มีอยู่

  7. 7
    Validate → Preview → Render

    รัน npx hyperframes lint → validate → preview ตรวจสอบใน browser → render --output final.mp4 ได้ไฟล์ MP4 พร้อมใช้

★ สาระสำคัญ: ขั้นตอนที่คนมักข้าม แต่สำคัญที่สุด คือ Step 1 (Plan) — Agent ที่รู้ narrative ก่อนเขียนโค้ดจะ loop น้อยกว่ามาก

กรณีใช้งานจริง

HyperFrames เหมาะกับงานแบบใด?

เหมาะมาก

Explainer Video

วิดีโออธิบายแนวคิดที่มีข้อความ, diagram เคลื่อนไหว, และ step-by-step animation — HyperFrames เขียน HTML ได้ตรงกว่า AI video tools เพราะ layout ควบคุมได้ pixel-perfect

  • Template แนะนำ: swiss-grid, kinetic-type
  • ใช้ GSAP animate ข้อความและ icon ทีละบรรทัด
  • เพิ่ม TTS voiceover ด้วย npx hyperframes tts

ตัวอย่าง Structure

Scene Layout

Scene 1: Hook (0–5s) → Scene 2: ปัญหา (5–15s) → Scene 3: โซลูชัน + demo (15–40s) → Scene 4: Call to Action (40–50s)

แต่ละ scene = 1 HTML section ที่มี data-start และ data-duration

เหมาะสำหรับ

Short-form (Reels / TikTok)

วิดีโอแนวตั้ง 9:16 ความยาว 15–60 วินาที พร้อม auto-caption และ beat-synced animation

  • Template แนะนำ: play-mode, vignelli (9:16)
  • ใช้ audio-reactive: bass → scale pulse, treble → glow
  • Caption อัตโนมัติด้วย npx hyperframes transcribe

ข้อดีเทียบ AI Video

ทำซ้ำได้ทันที

เปลี่ยนข้อความใน HTML → render ใหม่ → ได้วิดีโอใหม่ใน <2 นาที ไม่ต้อง re-prompt AI ทุกครั้ง เหมาะสำหรับ template-based content ที่ต้องออกบ่อยๆ

Advanced Use Case

Website Promo / Product Launch

ใช้ Three.js หรือ WebGL shader สำหรับ 3D Canvas effects วิดีโอโปรโมทที่ดูเหมือน motion design studio ผลิตให้ แต่ Agent เขียนโค้ดเอง

  • Template: product-promo
  • Shader transitions: cross-warp-morph, liquid-wipe
  • Website-to-video: capture URL เป็น video scene ได้ตรงๆ

Template System

50+ Prebuilt Templates

เปลี่ยนเนื้อหาในวิดีโอได้ทันทีโดยแค่แก้ HTML — ไม่ต้องตัดต่อใหม่ทั้งหมด เหมาะสำหรับ campaign ที่มีหลาย variant หรือหลายภาษา

★ สาระสำคัญ: HyperFrames เหนือ AI video tools เมื่อต้องการ consistency, repeatability, หรือ data-driven content — ไม่ใช่สำหรับวิดีโอที่ต้องการ "ความบังเอิญสร้างสรรค์"
06

ก่อนเริ่มใช้งาน

ข้อจำกัดและสิ่งที่ต้องรู้

สำหรับนักพัฒนาที่มี environment อยู่แล้วไม่ยาก แต่สำหรับมือใหม่ต้องติดตั้งเพิ่ม 3 อย่าง ขั้นต่ำ: node --version ต้อง ≥22, ffmpeg -version ต้องมี, และ Chrome Headless Shell ที่ Puppeteer จัดการให้อัตโนมัติ

Headless Chrome จะ seek ไปยัง frame ที่ต้องการ (ไม่ได้ play จริงๆ) — ถ้า animation มี infinite loop หรือ random values ผลลัพธ์จะแตกต่างกันในแต่ละ frame ทำให้ render เสีย ต้องใช้ finite, seekable timelines เท่านั้น

Remotion ใช้ React — ดีสำหรับทีมที่มี React codebase อยู่แล้ว HyperFrames ใช้ plain HTML — ดีกว่าสำหรับ AI agents เพราะ LLM เขียน HTML ได้ดีกว่า JSX ไม่ต้องมี bundler และ agent handoff ง่ายกว่ามาก

เหมาะ: animated charts, explainer videos, data-driven content, motion graphics, template-based social content, product demos ที่มีข้อความและ icon

ไม่เหมาะ: วิดีโอที่ต้องการ live-action footage, face generation, realistic camera movement — สำหรับงานเหล่านั้น AI video tools ยังดีกว่า

⚠ Gotcha ที่พบบ่อย: GSAP timeline ที่ไม่ได้ register บน window.__timelines จะไม่ถูก seek — animation จะไม่ทำงานใน render แม้ preview ดูปกติ
★ สาระสำคัญ: Animation ทุกตัวต้องเป็น seekable + finite + registered — เช็ค 3 ข้อนี้ก่อน render ทุกครั้ง

Key Takeaways

  • HTML = รูปแบบวิดีโอใหม่: HyperFrames จับ HTML frame-by-frame ผ่าน Headless Chrome → FFmpeg → MP4 ฟรี ไม่มี per-render fee
  • Deterministic ทุก render: input เดิม → output เหมือนกันทุก frame เหมาะสำหรับ CI/CD และ automated workflows
  • Agent-native design: HTML เป็น format ที่ LLM เขียนได้ดีที่สุด — Hermes + skill ทำงานได้ทันทีโดยไม่ต้องมี GUI
  • Template system ทรงพลัง: 50+ templates เปลี่ยนแค่ HTML content ได้วิดีโอ variant ใหม่ใน 2 นาที
  • ข้อจำกัดหลัก: ต้องการ Node 22+, FFmpeg, seekable animations — ไม่เหมาะกับ live-action หรือ face generation
  • เริ่มง่าย: npx skills add heygen-com/hyperframes แล้วบอก Agent ว่าอยากสร้างวิดีโอแบบไหน
