ทำไมต้องใช้ระบบต้นแบบไร้บัดกรี?

ระบบต้นแบบอิเล็กทรอนิกส์แบบไม่ต้องบัดกรีเปลี่ยนวิธีที่นักพัฒนาสร้างวงจรต้นแบบอย่างสิ้นเชิง แทนที่จะเสียเวลากับสายไฟที่หลุดง่าย ขาสั้นบน breadboard หรือการเดินสายที่ซับซ้อน ระบบเหล่านี้ใช้หัวต่อมาตรฐานที่เสียบแล้วล็อคอยู่กับที่ ทำให้การต่อเซนเซอร์หรือโมดูลใหม่ใช้เวลาไม่ถึงสิบวินาที และไม่มีความเสี่ยงที่จะต่อผิดขั้ว ผลลัพธ์คือนักพัฒนาสามารถโฟกัสกับการเขียนโค้ดและ logic ของงาน แทนที่จะมาแก้ปัญหาฮาร์ดแวร์ที่ไม่ควรเกิด

Section 1 — Deep-Dive Protocol · ส่วนประกอบ

ส่วนประกอบหลักที่ต้องเข้าใจ

ก่อนเลือกว่าจะใช้ระบบใด สิ่งสำคัญคือต้องเข้าใจว่าระบบต้นแบบไร้บัดกรีประกอบด้วยอะไรบ้าง ตั้งแต่ชั้น physical ของหัวต่อไปจนถึงชั้น protocol ที่ส่งข้อมูล และ ecosystem ของบอร์ดที่รองรับ ความเข้าใจในแต่ละชั้นช่วยให้ตัดสินใจเลือกระบบได้ถูกต้องตั้งแต่ต้น

ส่วนประกอบหลัก (Components)

Component 1

ปัญหา Breadboard

สายหลุด ขาสั้น ต่อผิด — เหตุผลที่ระบบ no-solder ถูกสร้างขึ้น

Component 2

Physical Layer — หัวต่อ JST

ขนาด pitch (1mm vs 2mm), จำนวนขา, กลไก locking ที่ป้องกันการดึงหลุด

Component 3

Protocol Stack

I2C, UART, Analog, Digital — แต่ละระบบรองรับสัญญาณต่างกัน

Component 4

Voltage Compatibility

3.3V vs 5V — จุดอันตรายหลักที่อาจทำ board เสียได้หากเชื่อมต่อผิด

Component 5

Ecosystem & Lock-in

จำนวนบอร์ดที่รองรับ, ผู้ผลิต, ราคา และความเป็น open standard

Component 6

I2C Address Management

การ daisy-chain ทำงานได้ถึง 127 devices ทางทฤษฎี แต่ address conflict คือปัญหาจริง

ปัญหา Breadboard → หัวต่อ JST มาตรฐาน → บัส I2C → Ecosystem ของบอร์ดและเซนเซอร์

★ Mental model: ระบบ no-solder คือ "หัวต่อมาตรฐาน + บัส I2C" ที่แก้ปัญหา breadboard ด้วยการ lock ขาไว้ไม่ให้หลุด

Section 1 — Deep-Dive Protocol · สมมติฐาน

สมมติฐานที่ซ่อนอยู่และสิ่งที่ต้องรู้ก่อน

สมมติฐานที่ผู้เริ่มต้นมักมองข้าม

สิ่งเหล่านี้คือสิ่งที่ระบบ no-solder ถือว่าผู้ใช้รู้อยู่แล้ว แต่มักไม่ถูกระบุอย่างชัดเจนในเอกสาร การเข้าใจก่อนจะช่วยหลีกเลี่ยงความผิดพลาดที่ทำ board เสียได้

  • สมมติฐานที่ 1 — STEMMA QT เหมือน Qwiic ทุกประการ: หัวต่อ JST SH 1mm 4 ขาของทั้งสองระบบเหมือนกัน ไม่ใช่แค่ "คล้ายกัน" — สายสามารถใช้แทนกันได้จริง แต่ชื่อ "STEMMA" (ไม่มี QT) คือระบบที่ต่างออกไปโดยสิ้นเชิงที่ใช้ JST PH 2mm
  • สมมติฐานที่ 2 — Grove รองรับ 5V แต่เซนเซอร์ส่วนใหญ่เป็น 3.3V: การเชื่อม Grove 5V เข้ากับ microcontroller 3.3V โดยไม่มี level shifter อาจทำให้ board เสียถาวร ต้องตรวจสอบ VCC ของเซนเซอร์ก่อนเสมอ
  • สมมติฐานที่ 3 — I2C address conflict เป็นปัญหาจริง: ถ้าซื้อเซนเซอร์ชนิดเดิมมาต่อเพิ่ม (เช่น temperature sensor สองตัว) ทั้งคู่จะมี address เดิม ต้องใช้ I2C multiplexer หรือเลือกรุ่นที่ปรับ address ได้
  • สมมติฐานที่ 4 — No-solder คือ prototype เท่านั้น: หัวต่อแบบ snap-in ไม่ออกแบบมาสำหรับการสั่นสะเทือน แรงดึง หรือสภาพแวดล้อมที่รุนแรง งาน production ต้องบัดกรีหรือทำ PCB จริงๆ

Takeaway 1 — Big Picture

ระบบ no-solder แก้ปัญหา breadboard ด้วยหัวต่อล็อค

ไม่ใช่แค่ "สะดวกขึ้น" แต่คือการกำจัดข้อผิดพลาดที่เกิดจากฮาร์ดแวร์ออกไปทั้งหมด ทำให้การ debug โฟกัสที่โค้ดได้เต็มที่

เลือก Qwiic/STEMMA QT สำหรับ I2C บน 3.3V

สำหรับ ESP32, RP2040, Arduino Nano 33 — Qwiic คือตัวเลือกที่ง่ายที่สุด มีบอร์ดให้เลือกมากและ library พร้อมใช้

Takeaway 3 — Grove Use Case

เลือก Grove เมื่อต้องการ 5V หรือ non-I2C

Takeaway 4 — Cross-System

STEMMA QT = Qwiic — สายใช้แทนกันได้เสมอ

ซื้อสายได้จาก SparkFun หรือ Adafruit ไม่ต้องยึดติดกับ brand เดียว ประหยัดได้มากเมื่อของหายหรือต้องการสายยาว

💡 Takeaways 5–7: (5) I2C address conflict คือปัญหาหลักเมื่อ daisy-chain เซนเซอร์ชนิดเดียวกัน — ใช้ i2c_scanner ก่อนซื้อเซนเซอร์ชุดที่สอง (6) Library ที่ใช้กับทั้ง 3 ระบบเหมือนกัน เพราะมันคือ I2C มาตรฐาน — Wire.h ใช้ได้ทุกระบบ (7) No-solder เหมาะกับ prototype เท่านั้น — production ต้องการ custom PCB หรือการบัดกรีเพื่อความทนทาน

Section 2 — ปัญหา Breadboard · สัญชาตญาณ

Breadboard ทำงานอย่างไร และทำไมถึงยังไม่พอ?

Breadboard คือเครื่องมือต้นแบบที่นักอิเล็กทรอนิกส์ใช้มาหลายทศวรรษ — เสียบสายแล้วถอดได้โดยไม่ต้องบัดกรี ฟังดูดี แต่ในทางปฏิบัติมีปัญหาหลายอย่างที่เพิ่มขึ้นตามขนาดและความซับซ้อนของโปรเจกต์ ทำให้การหาข้อผิดพลาดใช้เวลานานโดยไม่จำเป็น และทำให้โปรเจกต์ดูน่ากลัวสำหรับผู้เริ่มต้น

ปัญหาหลักของ Breadboard ที่พบในชีวิตจริง

Breadboard ทำงานได้ดีสำหรับวงจรง่ายๆ ที่มีชิ้นส่วนไม่กี่ตัว แต่เมื่อโปรเจกต์ซับซ้อนขึ้น ปัญหาเริ่มปรากฏ สาย jumper ที่ใช้เชื่อมต่อระหว่างชิ้นส่วนมีความหนาพอๆ กับขาของชิ้นส่วนที่ใส่ใน breadboard ทำให้เมื่อขยับหรือถือ board ขึ้นมา สายเหล่านั้นมักหลุดออกโดยไม่รู้ตัว

นอกจากนั้น การเดินสายใน breadboard ที่มีหลายส่วนประกอบพร้อมกันทำให้เกิด "สปาเก็ตตี้สาย" ที่มองยากว่าสายไหนต่อไปที่ไหน การตรวจสอบว่าสาย power, ground, SDA, SCL ต่อถูกต้องหรือเปล่าต้องนับขาทีละเส้น ซึ่งเป็นแหล่งของข้อผิดพลาดมากที่สุดในโปรเจกต์ฮาร์ดแวร์

ยิ่งไปกว่านั้น breadboard ออกแบบมาสำหรับ lab เท่านั้น ไม่ใช่สำหรับการใช้งานจริง การเคลื่อนย้าย board ในขณะที่โปรแกรมกำลังทำงานอยู่ทำให้ค่าเซนเซอร์กระโดดหรือโปรแกรม crash โดยไม่มีเหตุผลชัดเจน เสียเวลา debug ฝั่ง software ไปหลายชั่วโมง กว่าจะพบว่าจริงๆ แล้วสาย power หลุดออกครึ่งหนึ่ง

สายแบบ breadboard (ซ้าย) หลุดง่ายและดูยาก vs หัวต่อแบบ snap-in (ขวา) ที่ lock อยู่กับที่

★ Beginner takeaway: ถ้า bug หายเองเมื่อแตะสาย — ปัญหาคือ breadboard ไม่ใช่โค้ด

Section 2 — ปัญหา Breadboard · กลไก

ระบบ No-Solder แก้ปัญหาอย่างไร?

แนวคิดหลักของระบบ no-solder คือการแทนที่สาย jumper ที่หลุดง่ายด้วยหัวต่อมาตรฐานที่มีกลไก locking — เสียบแล้วดึงไม่ออกโดยไม่ตั้งใจ สายแต่ละเส้นมีหัวต่อที่ถูกกำหนด pinout ไว้แล้ว ทำให้ต่อผิดขั้วไม่ได้ เพราะหัวต่อมีทิศทางเดียวที่ใส่ได้

กลไก Keyed Connector — ต่อผิดไม่ได้

หัวต่อ JST ที่ใช้ในระบบ Qwiic และ STEMMA QT มีรูปร่างที่ไม่สมมาตร ทำให้ใส่ได้ทิศทางเดียวเท่านั้น ไม่มีทางใส่กลับหัวได้ ซึ่งหมายความว่า VCC (3.3V) ไม่มีทางต่อเข้ากับ GND โดยบังเอิญ และ SDA ไม่มีทางเข้าช่อง SCL การต่อผิดที่พบบ่อยใน breadboard จึงถูกกำจัดออกไปโดยสิ้นเชิง

นอกจากนั้น หัวต่อ JST ยังมีกลไก friction fit ที่ทำให้สายยึดอยู่กับที่โดยไม่หลุดแม้จะเขย่า board การถอดสายต้องดึงด้วยแรงพอสมควร ไม่ใช่แค่สัมผัสเบาๆ เหมือน jumper wire ที่หลุดได้เอง

เจาะลึก: ทำไม 4 ขาถึงพอสำหรับ I2C?

I2C ใช้เพียงสองสายสัญญาณคือ SDA (Serial Data) และ SCL (Serial Clock) ทำให้รวมกับ VCC และ GND แล้วได้ 4 ขาพอดี นี่คือเหตุผลที่ทุกระบบ no-solder ที่เน้น I2C ใช้ connector 4 ขา บัสสองเส้นนี้รองรับ device ได้ถึง 127 ตัวบน bus เดียวกัน (ทางทฤษฎี) โดย MCU เป็น master และ sensor แต่ละตัวมี address 7-bit ที่ไม่ซ้ำกัน

Pipeline: จาก Prototype สู่งานถาวร

ระบบ no-solder ออกแบบมาสำหรับ phase แรกของการพัฒนา ไม่ใช่ผลิตภัณฑ์สุดท้าย workflow ที่ดีคือ: เริ่มด้วย no-solder เพื่อ test concept → ยืนยันว่า logic และ sensor ทำงานได้ → ย้ายไป custom PCB หรือบัดกรีสำหรับ production

  1. 1
    Prototype ด้วย No-Solder

    ต่อ sensor กับ MCU ผ่าน Qwiic/Grove เขียนโค้ดทดสอบ ปรับ algorithm — ไม่มี hardware failure เข้ามารบกวน

  2. 2
    Verify ว่า Concept ใช้งานได้

    ทดสอบใน use case จริง ตรวจ latency, accuracy, power consumption ก่อนลงทุนใน PCB

  3. 3
    ย้ายไป Custom PCB หรือบัดกรี

    เมื่อ design ยืนยันแล้ว ใช้ schematic เดิมออกแบบ PCB สำหรับ production ที่ทนทานกว่า

★ สาระสำคัญ: No-solder = prototype phase; custom PCB = production phase — อย่าสับสนระหว่างสองขั้นตอนนี้

Section 3 — ระบบ Qwiic · สัญชาตญาณ

Qwiic — ระบบ I2C แบบ snap-in จาก SparkFun

Qwiic เป็นระบบ no-solder ที่ SparkFun พัฒนาขึ้นในปี 2017 โดยมีเป้าหมายเดียวคือทำให้การเชื่อมต่อ I2C sensor ง่ายที่สุดเท่าที่จะเป็นไปได้ ระบบนี้ใช้หัวต่อ JST SH ขนาด 1mm ที่มีเพียง 4 ขา รองรับเฉพาะ I2C ที่แรงดัน 3.3V เท่านั้น ความเรียบง่ายนี้คือจุดแข็งที่ทำให้ ecosystem เติบโตได้เร็วมากจนมีบอร์ดรองรับกว่า 1,000 ชนิดในปัจจุบัน

Pinout และข้อกำหนดทางเทคนิค

หัวต่อ Qwiic มี 4 ขาเรียงตามลำดับ: ขาที่ 1 คือ GND (สายดำ), ขาที่ 2 คือ VCC 3.3V (สายแดง), ขาที่ 3 คือ SDA หรือ I2C Data (สายน้ำเงินหรือเขียว) และขาที่ 4 คือ SCL หรือ I2C Clock (สายเหลืองหรือขาว) pinout นี้เป็นมาตรฐานเดียวกันในทุกบอร์ดที่มีโลโก้ Qwiic ไม่ว่าจะเป็นของ SparkFun หรือผู้ผลิตอื่นที่ร่วมใช้มาตรฐานนี้

SparkFun ออกแบบ Qwiic ให้เป็น open standard — ไม่มีค่า royalty ผู้ผลิตอื่นสามารถใช้โลโก้และมาตรฐาน Qwiic ได้ฟรี นโยบายนี้ทำให้ ecosystem เติบโตเร็วและราคาบอร์ดถูกลงเพราะมีการแข่งขัน

Qwiic Specs

ข้อกำหนดทางเทคนิค

  • Connector: JST SH 1.0mm pitch, 4-pin
  • Protocol: I2C เท่านั้น
  • Voltage: 3.3V (ห้ามใช้กับ 5V logic โดยตรง)
  • Speed: I2C standard (100 kHz) หรือ fast (400 kHz)
  • Max devices: 127 (ทางทฤษฎี)

Qwiic Ecosystem

บอร์ดและ Sensor

  • Sensors: BME280, APDS9960, VL53L1X, GPS, IMU
  • Displays: OLED, LCD character, e-ink
  • MCU boards: ESP32, RP2040, Arduino Nano 33
  • ผู้ผลิตร่วม: Adafruit (STEMMA QT), DFRobot, หลายราย

Daisy-Chain — ต่อ sensor หลายตัวบน bus เดียว

จุดเด่นของ I2C คือสามารถต่อ sensor หลายตัวบน bus เดียวกันโดยแต่ละตัวมี address ต่างกัน บอร์ด Qwiic ส่วนใหญ่มีหัวต่อ 2 อัน (ขาเข้าและขาออก) เพื่อให้ daisy-chain ได้โดยตรงโดยไม่ต้องใช้ hub เพิ่มเติม ลักษณะนี้ทำให้การเพิ่ม sensor ใหม่ง่ายมาก เพียงแค่เสียบสายต่อท้ายตัวสุดท้ายในสาย

อย่างไรก็ตาม หาก sensor สองตัวมี I2C address เดียวกัน (เช่น BME280 สองตัวที่มี address 0x76 หรือ 0x77 เหมือนกัน) จะทำงานด้วยกันบน bus เดียวไม่ได้ ต้องใช้ I2C multiplexer (เช่น TCA9548A) เพื่อแยก bus หรือซื้อรุ่นที่สามารถ set address ต่างกันได้

★ Beginner takeaway: Qwiic คือตัวเลือกที่ดีที่สุดสำหรับมือใหม่ที่ใช้ ESP32 หรือ Arduino 3.3V — เสียบแล้วทำงานได้ทันที

Section 3 — ระบบ Qwiic · กลไก

การใช้งาน Qwiic ในทางปฏิบัติ

การเริ่มต้นใช้ Qwiic ไม่ซับซ้อน — เสียบสาย เปิด library manager ใน Arduino IDE หรือ PlatformIO แล้วติดตั้ง library ของ sensor นั้นๆ ส่วนใหญ่ library จะ handle I2C communication ให้หมดโดยอัตโนมัติ ผู้พัฒนาเพียงแค่เรียก function ของ library เพื่ออ่านค่า sensor

ตัวอย่าง: ตรวจสอบ I2C Address ด้วย i2c_scanner

ก่อนเริ่มเขียน code สำหรับ sensor จริง ควรรันโปรแกรม i2c_scanner เพื่อตรวจสอบว่า MCU มองเห็น sensor ที่ต่ออยู่บน bus หรือไม่ และ address ที่ sensor ใช้จริงคืออะไร โปรแกรมนี้ scan address ตั้งแต่ 0x01 ถึง 0x7F และรายงานว่ามี device ตอบสนองที่ address ใดบ้าง

// i2c_scanner — วาง sketch นี้ใน Arduino IDE แล้ว upload
#include <Wire.h>

void setup() {
  Wire.begin();
  Serial.begin(115200);
  Serial.println("Scanning I2C bus...");
}

void loop() {
  byte nDevices = 0;
  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    byte error = Wire.endTransmission();
    if (error == 0) {
      Serial.print("Found device at 0x");
      if (addr < 16) Serial.print("0");
      Serial.println(addr, HEX);
      nDevices++;
    }
  }
  if (nDevices == 0) Serial.println("No I2C devices found");
  delay(3000);
}

โปรแกรมด้านบนใช้ Wire.beginTransmission(addr) เพื่อส่ง start signal ไปยัง address นั้น และ Wire.endTransmission() จะคืนค่า 0 หากมี device ตอบสนอง ถ้าเห็น 0x76 หรือ 0x77 แสดงว่า BME280 ต่อถูกต้อง; ถ้าไม่เห็นอะไร ให้ตรวจสอบว่าแรงดันไฟ 3.3V จ่ายให้ board แล้วหรือยัง

การอ่านค่า Sensor จริง — BME280 ตัวอย่าง

หลังจาก verify ด้วย i2c_scanner แล้ว สามารถติดตั้ง library ของ sensor นั้นๆ ผ่าน Arduino Library Manager (ค้นหา "BME280" แล้วเลือก library จาก Adafruit หรือ SparkFun) แล้วเรียกใช้ function ที่ library จัดเตรียมไว้

#include <Wire.h>
#include <Adafruit_BME280.h>

Adafruit_BME280 bme;

void setup() {
  Serial.begin(115200);
  Wire.begin();  // SDA, SCL ถูก map อัตโนมัติตาม board
  bme.begin(0x76);  // หรือ 0x77 แล้วแต่ address ที่ i2c_scanner เห็น
}

void loop() {
  float temp = bme.readTemperature();
  float humidity = bme.readHumidity();
  float pressure = bme.readPressure() / 100.0F;
  Serial.printf("Temp: %.1f C | Humidity: %.1f%% | Pressure: %.0f hPa\n",
                temp, humidity, pressure);
  delay(2000);
}

Wire.begin() ไม่ต้องระบุ pin เพราะ library จะ map SDA/SCL ตาม board definition ที่ติดตั้งใน Boards Manager โดยอัตโนมัติ สำหรับ ESP32 โดยทั่วไปคือ GPIO21 (SDA) และ GPIO22 (SCL) แต่ตรวจสอบ pinout ของ board ที่ใช้ก่อนเสมอ

เจาะลึก: กำหนด SDA/SCL ใน Wire.begin() เองเมื่อใด?

ใน ESP32 สามารถกำหนด pin ที่ต้องการใช้ได้ด้วย Wire.begin(SDA_PIN, SCL_PIN) เช่น Wire.begin(21, 22) ซึ่งมีประโยชน์เมื่อ board นั้นๆ ไม่ได้มี Qwiic connector ในตัว และต้องใช้ adapter cable ไปยัง pin อื่น หรือเมื่อต้องการสร้าง second I2C bus สำหรับ device ที่มี address ชนกัน: Wire1.begin(16, 17)

★ สาระสำคัญ: รัน i2c_scanner ก่อนเสมอ — ตรวจสอบ address และยืนยัน sensor ต่อถูกก่อนเขียนโค้ดจริง

Section 4 — ระบบ Grove · สัญชาตญาณ

Grove — ระบบต้นแบบที่เก่าแก่ที่สุดและ ecosystem ใหญ่ที่สุด

Grove คือระบบ no-solder ที่ Seeed Studio พัฒนาขึ้นในปี 2010 ซึ่งเก่ากว่า Qwiic ถึง 7 ปี ด้วยระยะเวลาที่ยาวนานนี้ Grove จึงมี ecosystem ที่ใหญ่ที่สุดในบรรดาทั้งสามระบบ และมีจุดเด่นสำคัญที่ Qwiic ทำไม่ได้คือรองรับสัญญาณได้หลายประเภท ทั้ง I2C, UART, Analog และ Digital รวมถึงรองรับแรงดันไฟทั้ง 3.3V และ 5V

4 ประเภทของ Grove Connector

Grove ใช้หัวต่อ 4 ขาขนาด 2.0mm pitch เหมือนกันทุกรุ่น แต่สิ่งที่แตกต่างคือ protocol ที่วิ่งผ่านสายแต่ละเส้น ซึ่งแบ่งเป็น 4 ประเภทหลัก ผู้ใช้ต้องดูที่ตัว sensor หรือ module ว่าเป็นประเภทใดก่อนซื้อ เพราะถ้าต่อ Grove-Analog เข้ากับ port I2C บน MCU อาจไม่ทำงาน

Grove-I2C

Type 3

Grove-Analog

Type 4

ขา: D0 (เหลือง), D1 (ขาว), VCC (แดง), GND (ดำ) — ใช้กับ button, relay, LED module, ultrasonic range finder

จุดเด่นของ Grove เหนือ Qwiic

Grove รองรับ 5V ซึ่งหมายความว่าใช้งานร่วมกับ Arduino Uno, Mega, Leonardo ได้โดยตรงโดยไม่ต้องกังวลเรื่องแรงดัน นอกจากนั้น ecosystem ที่ใหญ่กว่า Qwiic มาก (Seeed Studio มีผลิตภัณฑ์ Grove มากกว่า 300 รายการ) ทำให้หา sensor ชนิดที่ต้องการได้ง่ายกว่า โดยเฉพาะ sensor ที่ผิดแผกจากมาตรฐาน เช่น gas sensor, soil moisture, weight sensor

★ Beginner takeaway: ถ้าใช้ Arduino Uno 5V หรือต้องการ sensor ที่ไม่ใช่แค่ I2C — Grove คือตัวเลือกที่ดีกว่า

Section 4 — ระบบ Grove · กลไก

การใช้งาน Grove ในทางปฏิบัติ: แรงดันและ Protocol

การใช้งาน Grove อย่างถูกต้องต้องเข้าใจ 2 เรื่องหลัก: ประเภทของ port ที่ MCU มีให้ (I2C / Analog / Digital / UART) และแรงดันไฟ (3.3V หรือ 5V) ของทั้ง MCU และ sensor ที่ต้องการเชื่อมต่อ การมองข้ามขั้นตอนนี้คือสาเหตุที่ทำให้ board เสียได้มากที่สุดในระบบ Grove

การตรวจสอบแรงดันไฟก่อนเชื่อมต่อ — Critical Step

ก่อนต่อ Grove sensor เข้ากับ MCU ต้องตรวจสอบ 2 จุดเสมอ: แรงดัน VCC ของ MCU board ที่ output ออกมา (3.3V หรือ 5V) และแรงดัน logic ของ sensor นั้นๆ ที่ระบุใน datasheet ถ้า MCU ให้ VCC เป็น 3.3V แต่ sensor ต้องการ 5V sensor อาจทำงานได้ไม่ถูกต้อง ในทางกลับกัน ถ้า sensor ให้สัญญาณ 5V กลับมาที่ MCU ที่รองรับแค่ 3.3V logic — MCU pin อาจเสียทันที

วิธีที่ปลอดภัยที่สุดเมื่อต้องผสม 3.3V และ 5V คือใช้ logic level converter ที่มี Grove connector สำเร็จรูปอยู่แล้ว เพียงต่อสายระหว่าง MCU กับ converter แล้วต่อ converter กับ sensor โดยไม่ต้องดัดแปลงอะไรเพิ่มเติม

เจาะลึก: ทำไม Arduino Uno 5V ถึงทำ ESP32 3.3V เสียได้?

Arduino Uno ใช้ Atmega328P ที่ทำงานที่ 5V logic ดังนั้น digital output จาก Uno คือสัญญาณ 5V ESP32 ทนแรงดันที่ GPIO pin ได้สูงสุดเพียง 3.6V การส่งสัญญาณ 5V เข้า GPIO ของ ESP32 จะทำให้ ESD protection diode ภายในชิปทำงานเกินพิกัดและเสียถาวรภายในไม่กี่วินาที ต้องใช้ level shifter เสมอเมื่อเชื่อม Grove จาก Uno เข้ากับ ESP32

Grove Base Shield — วิธีง่ายที่สุดสำหรับ Arduino

Seeed Studio ผลิต Grove Base Shield ซึ่งเป็น expansion board ที่เสียบเข้ากับ Arduino Uno หรือ Mega โดยตรง แล้วให้ Grove connector ในรูปแบบต่างๆ มาถึง 16 port (I2C, Analog, Digital, UART) ทำให้ไม่ต้องใช้สายแปลง Breadboard และจัดการ power rail ให้อัตโนมัติ

Section 5 — STEMMA & STEMMA QT · สัญชาตญาณ

STEMMA QT — ระบบจาก Adafruit ที่เข้ากันได้กับ Qwiic

Adafruit พัฒนาระบบ STEMMA ขึ้นในปี 2014 โดยมี 2 variant ที่ต้องแยกแยะให้ถูกต้อง คือ STEMMA (ใหญ่, 2mm) และ STEMMA QT (เล็ก, 1mm, เหมือน Qwiic) ความสับสนจาก naming นี้เป็นสาเหตุหลักที่ทำให้ผู้เริ่มต้นซื้อสายผิด ความเข้าใจที่ถูกต้องคือ STEMMA QT คือ Qwiic ในชื่อของ Adafruit — ใช้สายเดียวกันได้ทันที

สองระบบในชื่อเดียวกัน: STEMMA vs STEMMA QT

STEMMA ดั้งเดิม (ไม่มี QT) ใช้หัวต่อ JST PH ขนาด 2.0mm ซึ่งใหญ่กว่าและเสียบง่ายกว่าสำหรับผู้ที่มีมือใหญ่หรือทำงานกับชิ้นส่วนขนาดใหญ่ ระบบนี้ยังรองรับทั้ง I2C และสัญญาณ analog ด้วย connector 3 ขา (GND, VCC, Signal) ทำให้ใช้กับ servo, potentiometer และ sensor analog ทั่วไปได้

STEMMA QT (ชื่อเล่น "Cutie") คือระบบที่ Adafruit เพิ่มเข้ามาเพื่อรองรับ I2C โดยเฉพาะ โดยใช้หัวต่อ JST SH ขนาด 1.0mm 4 ขา — ขนาดเดียวกับ Qwiic ทุกประการ Adafruit ประกาศอย่างชัดเจนว่า STEMMA QT และ Qwiic ใช้สายแทนกันได้ 100% ทำให้ผู้ใช้ไม่ต้องซื้อสายแยกกันสำหรับสองระบบนี้

STEMMA (ไม่มี QT)

JST PH 2.0mm

  • 4-pin: I2C (SCL/SDA/VCC/GND)
  • 3-pin: Analog/Digital (GND/VCC/Signal)
  • รองรับ 3.3V และ 5V
  • ไม่เข้ากับ Qwiic

STEMMA QT

  • 4-pin: GND/VCC/SDA/SCL (ลำดับเดียวกับ Qwiic)
  • รองรับ 3.3V (และ 5V tolerant บางบอร์ด)
  • สายใช้แทนกับ Qwiic ได้ทันที
  • Adafruit ยืนยันว่า fully compatible

ทำไม Adafruit ถึงสร้าง STEMMA QT ซ้อนกับ Qwiic?

ความแตกต่างที่แท้จริงไม่ได้อยู่ที่หัวต่อ แต่อยู่ที่ ecosystem และ library — Adafruit มี library ชื่อดัง อย่าง Adafruit Unified Sensor, CircuitPython และ Arduino library ที่รองรับ sensor ของตัวเองได้ดีกว่า SparkFun ในหลายรุ่น การมี STEMMA QT ทำให้ Adafruit สามารถเพิ่มหัวต่อนี้ในบอร์ดของตัวเองเพื่อขายร่วมกับ sensor ในระบบ Adafruit โดยยังคง interoperability กับ SparkFun Qwiic ไว้ด้วย

★ Beginner takeaway: STEMMA QT = Qwiic สายสามารถใช้แทนกัน; STEMMA (ไม่มี QT) = ระบบต่างหาก ต้องใช้ adapter

Section 5 — STEMMA & STEMMA QT · ความเข้ากันได้

Cross-System Compatibility — ผสมระหว่าง Qwiic, Grove และ STEMMA

ในโปรเจกต์จริง มักพบว่า sensor บางตัวมีแค่ Grove connector ขณะที่ MCU มี Qwiic หรือ STEMMA QT ข่าวดีคือมี adapter cable และ breakout board ที่ช่วยเชื่อมระหว่างระบบเหล่านี้ได้ ข่าวสำคัญคือต้องระวังเรื่องแรงดันไฟเมื่อผสมระบบ เพราะ Qwiic/STEMMA QT เป็น 3.3V เท่านั้น ขณะที่ Grove บางรุ่นเป็น 5V

Adapter Options ที่หาซื้อได้

SparkFun ผลิต adapter ที่แปลง Qwiic ไปเป็น Grove (4-pin I2C) และในทิศทางกลับ Adafruit ก็มี STEMMA QT to Grove adapter เช่นกัน นอกจากนั้นยังมี breakout cable ที่แปลงหัวต่อ Qwiic หรือ Grove เป็น DuPont pin header ธรรมดา ทำให้นำไปเสียบกับ breadboard หรือ custom PCB ได้

สำหรับการผสมระหว่าง 3.3V และ 5V โดยเฉพาะ ควรซื้อ Adafruit 4-Channel I2C Level Shifter ที่มีหัวต่อ STEMMA QT ในตัว ซึ่งแปลงสัญญาณ 5V จาก Arduino Uno ลงมาเป็น 3.3V สำหรับ sensor ที่ต้องการแรงดันต่ำกว่า

เจาะลึก: ทำไม STEMMA QT ถึงบอกว่า "5V tolerant" บางบอร์ด?

บอร์ด Adafruit บางรุ่น (เช่น breakout sensor ที่ออกแบบสำหรับ Arduino Uno) มีวงจร level shifter หรือ voltage regulator ติดตั้งมาในตัวบอร์ด ทำให้รับ VCC 5V จาก Arduino ได้และแปลงลงเป็น 3.3V สำหรับ IC ภายใน สัญญาณ I2C ก็ถูก pull-up ไปที่ 3.3V โดยอัตโนมัติผ่าน resistor บนบอร์ด แต่สิ่งนี้ต้องอ่านจาก datasheet ของบอร์ดนั้นๆ ไม่ใช่สมมติเอาเอง

★ สาระสำคัญ: Qwiic ↔ STEMMA QT ใช้สายแทนกันได้เลย; Qwiic ↔ Grove I2C ต้องใช้ adapter; อย่าผสม 5V Grove กับ 3.3V Qwiic โดยไม่มี level shifter

Section 6 — การเลือกระบบ

เลือกระบบไหน? — Framework การตัดสินใจ

การเลือกระบบ no-solder ที่เหมาะสมขึ้นอยู่กับ 3 ปัจจัยหลัก: แรงดันไฟของ MCU ที่ใช้, ประเภทของสัญญาณที่ sensor ต้องการ และขนาดของ ecosystem ที่ต้องการ Framework ด้านล่างช่วยให้ตัดสินใจได้ตรงจุดโดยไม่ต้องอ่าน datasheet ของทุกระบบก่อน

แนะนำ: Qwiic หรือ STEMMA QT

ถ้าใช้ ESP32, RP2040, Arduino Nano 33, Feather หรือ board ใดก็ตามที่ทำงานที่ 3.3V และต้องการต่อ sensor ผ่าน I2C — Qwiic หรือ STEMMA QT คือตัวเลือกที่ดีที่สุด เหตุผลคือระบบทั้งสองออกแบบมาสำหรับ use case นี้โดยเฉพาะ ไม่ต้องกังวลเรื่องแรงดัน ไม่ต้องใช้ adapter และมี library พร้อมใช้สำหรับ sensor ยอดนิยมทุกรุ่น

เลือก Qwiic ถ้าต้องการซื้อของจาก SparkFun หรือ ecosystem ที่กว้างกว่า เลือก STEMMA QT ถ้าต้องการ sensor จาก Adafruit หรือต้องการ library จาก CircuitPython ทั้งสองใช้สายแทนกันได้ ดังนั้นเริ่มจากระบบใดก็ได้และผสมได้ตามต้องการ

เหมาะที่สุด: ESP32, RP2040, Arduino Nano 33 IoT/BLE, Adafruit Feather, Sparkfun Thing Plus, BBC micro:bit (ด้วย adapter)

แนะนำ: Grove พร้อม Base Shield

ถ้าใช้ Arduino Uno, Mega, Leonardo หรือ board 5V อื่นๆ หรือต้องการ sensor ที่ไม่ใช่ I2C (เช่น potentiometer, button, relay, ultrasonic, UART GPS) — Grove คือตัวเลือกที่ตรงที่สุด Grove Base Shield เสียบเข้ากับ Arduino ได้ทันทีโดยไม่ต้องซื้ออะไรเพิ่ม และมี port ครบทั้ง I2C, Analog, Digital และ UART

ข้อเสียของ Grove สำหรับผู้ใช้มือใหม่คือต้องระวังประเภทของ port (อย่าเสียบ Grove-Analog เข้า port I2C) และต้องตรวจสอบแรงดันของ sensor ก่อนใช้กับ board 3.3V

เหมาะที่สุด: Arduino Uno, Mega, งานที่ต้องการ non-I2C sensor, โรงเรียนและ education kit ที่ใช้ Grove ecosystem อยู่แล้ว

แนะนำ: เลือก Qwiic เป็นหลัก + adapter สำหรับ Grove

ถ้าโปรเจกต์มี MCU หลายตัวที่แรงดันต่างกัน หรือต้องการเชื่อม sensor จากหลาย ecosystem — วิธีที่ดีที่สุดคือเลือก Qwiic หรือ STEMMA QT เป็น primary system สำหรับ I2C sensor ทั้งหมด แล้วใช้ Grove-to-Qwiic adapter เมื่อจำเป็นต้องต่อ Grove sensor

วิธีนี้ประหยัด adapter ได้มากกว่าการผสมทั้งสองระบบเท่าๆ กัน เพราะ adapter ราคาถูกและมีขายจาก SparkFun และ Adafruit โดยตรง การมี Qwiic/STEMMA QT เป็น standard ของทีมยังทำให้การแชร์ sensor ระหว่างโปรเจกต์ง่ายขึ้นด้วย

เหมาะที่สุด: ทีมที่มีหลายโปรเจกต์, maker space, lab ที่มี sensor หลากหลายระบบ

Closing — Key Takeaways

สิ่งที่ต้องจำและขั้นตอนถัดไป

หลังอ่านบทเจาะลึกนี้จบ ผู้อ่านควรสามารถเลือกระบบ no-solder ที่เหมาะกับโปรเจกต์ เชื่อมต่อ sensor ได้ถูกต้อง และหลีกเลี่ยงข้อผิดพลาดที่พบบ่อยที่สุดในการใช้งาน Qwiic, Grove และ STEMMA QT

Key Takeaways

  • ระบบ no-solder กำจัดข้อผิดพลาดจากฮาร์ดแวร์: หัวต่อแบบ keyed ป้องกันการต่อผิดขั้ว และกลไก locking ป้องกันสายหลุด ทำให้ debug โฟกัสที่โค้ดได้เต็มที่
  • STEMMA QT = Qwiic ทุกประการ: สายสามารถใช้แทนกันระหว่าง Adafruit และ SparkFun ไม่ต้องซื้อสายสองชุดสำหรับสองระบบนี้
  • Qwiic/STEMMA QT สำหรับ I2C 3.3V; Grove สำหรับที่เหลือ: ถ้าใช้ ESP32 หรือ RP2040 และต้องการแค่ I2C — Qwiic ง่ายที่สุด; ถ้าต้องการ 5V หรือ non-I2C — Grove รองรับได้
  • ตรวจ VCC ก่อนต่อสายเสมอ: 5V เข้า GPIO 3.3V ทำให้ board เสียทันที ใช้ level shifter เมื่อผสมแรงดัน
  • รัน i2c_scanner ก่อน write code จริง: ยืนยันว่า MCU มองเห็น sensor และ address ถูกต้องก่อนลงทุนเวลาเขียน library
  • Address conflict แก้ด้วย I2C multiplexer: ถ้าต้องการ sensor ชนิดเดิมสองตัว ใช้ TCA9548A หรือเลือก sensor ที่ปรับ address ได้
  • No-solder คือ prototype phase: เมื่อ design ยืนยันแล้ว ย้ายไป custom PCB หรือบัดกรีเพื่อความทนทานใน production

Glossary

JST
Japan Solderless Terminal — ผู้ผลิตหัวต่อขนาดเล็กที่ใช้ใน Qwiic/STEMMA QT (SH 1mm) และ Grove (2mm)
I2C
Inter-Integrated Circuit — บัสสื่อสารแบบ 2 สาย (SDA + SCL) ที่ช่วยให้ MCU ต่อ sensor ได้หลายตัวพร้อมกัน
SDA / SCL
Serial Data / Serial Clock — สองสายของ I2C bus ที่ทุกระบบ no-solder แบบ I2C ใช้ร่วมกัน
Daisy-chain
การต่อ device หลายตัวเป็นสายโซ่บน bus เดียวกัน — ทำได้เพราะ I2C ใช้ address แยกแต่ละ device
Address conflict
ปัญหาที่เกิดเมื่อ sensor สองตัวมี I2C address เดียวกัน — MCU ไม่สามารถแยกแยะได้ว่าข้อมูลมาจากตัวไหน
Level shifter
วงจรที่แปลงแรงดันสัญญาณระหว่าง 5V และ 3.3V — จำเป็นเมื่อผสม board ต่างแรงดันบน I2C bus เดียวกัน
Keyed connector
หัวต่อที่มีรูปร่างไม่สมมาตร ทำให้ใส่ได้ทิศทางเดียว ป้องกันการต่อผิดขั้ว
I2C Multiplexer
IC (เช่น TCA9548A) ที่แยก I2C bus ออกเป็นหลาย channel ทำให้ต่อ sensor address เดียวกันได้หลายตัว
★ ถ้าจำได้อย่างเดียว: เลือกระบบตาม MCU voltage ก่อน → Qwiic/STEMMA QT สำหรับ 3.3V I2C; Grove สำหรับ 5V หรือ non-I2C — แล้วตรวจ address ก่อน deploy
